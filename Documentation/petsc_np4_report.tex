\section*{Scope}
This report documents the requested workflow:
\begin{itemize}
\item Build OGS in Release with PETSc, OpenMP, and MFront enabled.
\item Auto-generate \texttt{-np4} variants from existing \texttt{mpirun -np 1} tests.
\item Execute baseline (\texttt{petsc\_np1\_source}) and generated (\texttt{petsc\_np4\_variant}) test slices.
\item Compare outcomes pairwise and record reproducibility/revert notes.
\end{itemize}

\section*{Step Log (Reversible Logical Steps)}
\subsection*{Step 1: Preflight environment}
\begin{itemize}
\item Tools available: \texttt{mpicc}, \texttt{mpic++}, \texttt{mpirun}, \texttt{pkg-config}, \texttt{python3}.
\item Python: \texttt{3.13.12}; MPI launcher: MPICH Hydra \texttt{4.3.2}.
\item Local runtime workaround for this host during ctest execution:
\texttt{HYDRA\_LAUNCHER=fork HYDRA\_IFACE=lo0}.
\end{itemize}

\subsection*{Step 2: Test-generation and mesh-preparation fixes}
Implemented and verified:
\begin{itemize}
\item Robust multiline \texttt{<mesh>} parsing in
\texttt{scripts/cmake/test/AddTest.cmake} and
\texttt{scripts/cmake/test/PreparePartitionedMesh.cmake}.
\item Correct handling of \texttt{<OpenGeoSysProjectDiff>} mesh replacements and mesh-source resolution.
\item np4 setup now partitions the full mesh set in one call (main mesh + related boundary meshes), keeping partition mapping consistent.
\item \texttt{petsc\_np1\_source}/\texttt{petsc\_np4\_variant} remain mutually exclusive and 1:1 mapped.
\item Added np4 eligibility guard for tiny meshes: primary mesh must have at least
\texttt{2*np+1} elements (for np4: 9 elements), preventing unstable tiny-mesh np4 clones.
\item Tiny meshes excluded by this guard in this run include
\texttt{cube\_1x1x1\_hex\_1e0},
\texttt{cube\_1x1x1\_hex\_1e1},
\texttt{square\_1x1\_quad\_1e0},
\texttt{single\_cube\_bulk}.
\end{itemize}

\subsection*{Step 3: Configure/build (Release PETSc + OpenMP + MFront)}
Configured with:
\begin{verbatim}

  --preset release-petsc \
  -DOGS_ENABLE_PETSC_NP4_VARIANTS=ON
\end{verbatim}
